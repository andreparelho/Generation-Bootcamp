1. O que são metodos HTTP?
	É um protocolo de REQUEST e REPONSE. É um protocolo que contém um conjunto de métodos (verbos) de requisão que indicam a acão a ser execudada.

2. Quais são os métodos HTTP mais utlizados atualmente?
	GET, POST, PUT, DELETE.
	
3. Quais são os dois principais métodos HTTP e qual a diferença entre eles?
	GET e POST. 	
	A diferença entre eles é a visibilidade. GET: é enviada como String junto a uma URL. POST: é encapsulada junto ao corpo da requisição e não pode ser vista.

4. O que são requisições HTTP e qual os tipos?
	São mensagens do cliente para o servidor e como resposta ele inicia uma ação. Os tipos são GET, POST, PUT, DELETE, TRACE, OPTIONS, PATCH, HEAD E CONNECT.

5. Como funciona uma requisição?
	O cliente envia uma mensagem para servidor com todos os tipos de requisitos pedidos para o servidor, e o servidor envia uma ação como resposta. A requisição
é divide em três partes. request line, headers e body.

	IPv6: é a versão 6 do protocolo de internet com padrão de 128bits que permite cerca de 340undecilhoes de endereços.

10. Faça um resume sobre os seguintes protocolos: http, tcp, ip, udp, smtp e ftp.
	http: É um protocolo que contém um conjunto de métodos (verbos) de requisão que indicam a acão a ser execudada.
	
	tcp: significa Transmission Control Protocol. É responsavel pelo envio de pacotes e garantia da entrega bem sucedida deles. 	

	ip: significa Internet Protocol, que é a identificador que permite que as informações sejam enviadas entre dispositivos de uma rede.

	udp: significa User Data Protocol, é um protocolo de comunicação usado para estabelecer conexões de baixa latência e tolêrancia a perdas entre app de internet.
o udp acelera as transmissões, permitindo a transferencia de dados seja realizada.

	smtp: significa Simple Mail Transfer Protocol, usado por servidores de email para enviar, receber e/ou retransmitir emails de saída entre remetente e destina
tário de email.

	ftp: é o protocolo de transferencia de arquivos. É a maneira de baixar, carregar ou transferir arquivos de local para o outro usando a internet.
